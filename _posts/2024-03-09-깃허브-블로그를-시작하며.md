# 블로그 제작 시 고려한 점

이전부터 기술 블로그를 운영해야겠다는 생각을 계속 해오고 있었고 드디어 시작 하기로 마음을 먹었다.

그래서 플랫폼을 정하려고 했는데 크게 3가지 정도 선택지가 있었다.

- velog
- tistory
- github pages

전부 무료로 블로그를 운영할 수 있으며 이 중에서 github pages 를 선택하기로 했다.

## 왜 Github Pages 를 선택하였는가?

Github Pages 의 장점은 아래와 같다.

- 트래픽 제한이 100GB 이지만 소프트 제한이다.
- 티스토리와 벨로그에 비해 커스터마이징이 자유롭다.
- 광고를 통한 수익 창출이 가능하다.
- 개발자 포트폴리오에 블로그를 넣을 시 좋은 인상을 줄 수 있다.

## 도메인 주소를 구매한 이유

Github Pages 는 기본적으로 *.github.io 도메인 주소를 제공한다.

근데 github.io 도메인을 사용하게 되면 나중에 플랫폼 변경 시 도메인 자체를 변경해야 되기 때문에 도메인을 직접 구매해서 앞으로 블로그를 운영하다가 변경해도 이 도메인으로 유지 하려고 한다.

또, 개인 도메인을 구매해서 프로젝트에 서브 도메인을 사용하여 내가 만든 프로젝트임을 쉽게 확인할 수 있다.

### 도메인을 구매한 곳

Cloudflare 는 장기간 도메인 사용 시 비용이 가장 저렴하다고 판단했기 때문에 (.dev 기준) 이 사이트에서 구매하기로 하였다.

또한 Cloudflare 는 안정적이기도 하고 평소에 ssl, cdn 등의 서비스를 자주 이용하고 있었기 때문에 익숙하기도 하였다.

## Github Pages 정적 사이트 생성기와 테마

정적 사이트 생성기는 Jekyll, 테마는 Chirpy를 선택했다.

가장 국내 관련 자료를 찾기 쉬웠고, Chirpy 테마를 선택한 이유는 코드를 클릭만으로 쉽게 복사할 수 있고 한글화 기능이 있었기 때문이다.
